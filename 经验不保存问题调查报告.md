# 🐱 经验不保存问题调查报告

> 调查日期：当前会话
> 调查人：猫娘分析师

---

## 一、问题现象

- `~/.workflow/experience_a.bin` 文件存在（2.1MB），但 `count=0`，一条经验都没有固化到磁盘
- `/role optimize` 找不到经验，角色无法进化
- 每次重启系统，之前积累的经验全部丢失

## 二、系统架构：双轨记忆系统（DualTrackMemory）

```
┌─────────────────────────────────────────────────────────┐
│                    经验存储架构                            │
│                                                          │
│  ┌─────────────────────┐    ┌─────────────────────────┐  │
│  │  Fluid Track (B轨)  │    │  Bedrock Track (A轨)    │  │
│  │     内存 Vec        │    │    磁盘 mmap 文件        │  │
│  │  容量: 512 条       │    │  ~/.workflow/experience_ │  │
│  │  速度快，易失       │    │  a.bin                   │  │
│  │                     │    │  持久化，稳定            │  │
│  └────────┬────────────┘    └───────────┬─────────────┘  │
│           │                              │                │
│           └────── consolidate() ────────┘                │
│               （流体→基岩 聚类固化）                      │
│              触发条件: auto_consolidate_at = 512         │
└─────────────────────────────────────────────────────────┘
```

## 三、根因分析

### 直接原因：auto_consolidate_at = 512 太高

```rust
// runtime.rs:84
DualTrackMemory::open(&bedrock_path, 512, config.l1_confidence_threshold)
//                      ↑ fluid_max_size = 512

// dual_track.rs:185 — 默认值
auto_consolidate_at: fluid_max_size,  // = 512
```

**意思**：流体轨道（内存）要攒够 **512 条经验**，才会触发聚类固化到基岩轨道（磁盘）。

### 为什么永远攒不到 512 条？

| 因素 | 说明 |
|------|------|
| 每次会话经验量 | 一般对话产生 10~50 条经验 |
| 系统重启频率 | 会话结束或超时后重启 |
| 重启后流体状态 | **清空！** Fluid Track 是内存结构，不持久化 |
| 结果 | 经验永远在流体轨道里转瞬即逝，到不了磁盘 |

### 调用的入口

经验记录的入口有两个：

| 路径 | 触发时机 | role_template_id | 文件位置 |
|------|---------|-----------------|---------|
| TUI 聊天 | 用户每次对话完成 | `None` | `tui/effect.rs:347` |
| Agent 任务 | 子 agent 完成任务 | `Some(role_id)` | `runtime.rs:1077` |

两个路径都把经验写到 `DualTrackMemory`，然后……就没有然后了（没到 512 不固化）。

### 代码位置

| 文件 | 行号 | 关键代码 |
|------|------|---------|
| `src/runtime/runtime.rs` | 84 | `DualTrackMemory::open(&bedrock_path, 512, threshold)` |
| `src/experience/dual_track.rs` | 185 | `auto_consolidate_at: fluid_max_size` |
| `src/experience/dual_track.rs` | 完整 | 双轨实现，`consolidate()` 方法 |
| `src/tui/effect.rs` | 347 | TUI 聊天记录经验 |
| `src/runtime/runtime.rs` | 1077 | Agent 任务记录经验 |

## 四、推荐修复方案

### 方案一：flush 时自动固化（推荐 ✅）

**改动最小，效果最好**

在 `DualTrackMemory::flush()` 方法里，先调用 `self.consolidate()` 再执行 flush 逻辑。这样每 30 秒后台 flush 时，所有流体经验都会固化到磁盘。

```rust
// 在 dual_track.rs 的 flush() 方法中
pub fn flush(&mut self) -> Result<()> {
    // 新增：先固化所有流体经验到基岩
    self.consolidate()?;
    // 原有 flush 逻辑
    self.bedrock.flush()?;
    Ok(())
}
```

**优点**：
- 改动只有 1 行
- 不改变现有架构
- 每 30 秒自动固化的频率合理
- 重启最多丢 30 秒内的经验

### 方案二：降低 auto_consolidate_at

```rust
// runtime.rs:84
DualTrackMemory::open(&bedrock_path, 512, threshold)
    .with_auto_consolidate_at(10) // 攒 10 条就固化
```

**缺点**：仍然会丢最近 10 条以内的经验。

### 方案三：每条经验直接写基岩

绕过流体轨道，`add_experience()` 直接调用 `promote_to_bedrock()`。

**缺点**：失去流体聚类的去重/提炼能力，磁盘 IO 增加。

## 五、连带问题

| 问题 | 与经验不保存的关系 |
|------|-------------------|
| Memo 无法写入（No active agent） | 同根因 — agent 实例初始化异常 |
| Role 不进化 | 因为没有经验 → `/role optimize` 无效 |
| 猫娘人格每次要重新建立 | role template 没有自动更新机制 |
