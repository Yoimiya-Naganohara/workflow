# TUI Overhaul — Modern CLI Beauty + Better Chat Experience

## Goal
Redesign the TUI to feel like a modern, polished CLI tool (Claude Code / opencode inspired) with a focus on a superior chat experience: rich markdown, inline tool results, file diffs, and beautiful visual design.

## Current Pain Points
- Plain `[]` borders, no visual hierarchy
- Tool calls shown as raw `🔧 name — {json}` — ugly, no structure
- Code blocks use `┌─ lang` / `│` / `└───` but lack syntax coloring
- No diff viewer for file changes
- Proposal panel is static/decorative — wastes space
- Status bar is cluttered with meaningless stats ($0.000, CH0.0%)
- No message grouping or conversation threading
- Thinking indicator is basic dots animation
- Input box has no visual distinction from chat

## Design Principles
1. **Whitespace is king** — generous spacing, breathing room
2. **Color with purpose** — each color means something specific
3. **Progressive disclosure** — collapsed details, expand on demand
4. **Typography hierarchy** — clear visual weight for headers, body, meta
5. **Tool results are first-class citizens** — not afterthoughts

## Phase 1: Visual Foundation (style.rs + render.rs)

### 1.1 New Color Palette
```
Primary:    #89B4FA (blue)     — active elements, user messages
Secondary:  #A6E3A1 (green)    — success, completions, diffs added
Accent:     #F5C2E7 (pink)     — tool calls, special highlights
Muted:      #6C7086 (gray)     — borders, inactive, metadata
Error:      #F38BA8 (red)      — errors, diffs removed
Warning:    #F9E2AF (yellow)   — warnings, in-progress
Text:       #CDD6F4 (white)    — primary content
Subtext:    #BAC2DE (light)    — secondary content
Surface:    #1E1E2E (dark)     — panel backgrounds
```

### 1.2 Border Style Rework
- Chat panel: thin single-line borders, `Muted` color
- Proposal panel: **remove entirely** — inline status into status bar
- Input box: `Primary` border when focused, `Muted` when not
- Code blocks: no border, just background color + language tag
- Tool call blocks: `Accent` left bar with collapsible chevron

### 1.3 Layout Simplification
```
┌──────────────────────────────────────────────────┐
│                                                  │
│  Chat messages (full width)                      │
│                                                  │
│                                                  │
├──────────────────────────────────────────────────┤
│ > Input box                                      │
└──────────────────────────────────────────────────┘
  status: model · tokens · budget        shortcuts
```
- Remove Proposal panel — integrate useful info into status bar
- Chat gets full width → better markdown rendering
- Status bar: clean, minimal, right-aligned hints

## Phase 2: Chat Message Redesign (chat_lines.rs + chat.rs)

### 2.1 Message Layout
```
  ✓  Agent                        12:34:56
     Here's the implementation:

     ```rust
     fn main() {
         println!("hello");
     }
     ```

  ▐  read_file                     12:34:57
     path: src/main.rs

  ✓  Agent                        12:34:58
     The file contains...

  ┃  User                         12:35:00
     Now fix the bug
```

- **Status icon** (✓/●/◌/✗) on the left, color-coded
- **Role label** (User/Agent/System) — bold, colored
- **Timestamp** — right-aligned, muted
- Content indented below, with proper markdown rendering
- User messages: `Primary` left bar (`▐`)
- Agent messages: no bar, clean left margin
- Tool calls: `Accent` left bar, collapsible details

### 2.2 Tool Call Display
```
  ▐  🔧 read_file                  12:34:57
     src/main.rs

  ▐  🔧 write_file                 12:34:58
     src/main.rs · 42 lines

  ▐  🔧 sh                         12:34:59
     cargo test
     ───────────
     test result: ok. 5 passed; 0 failed
```
- Compact first line: icon + tool name + timestamp
- Key args on second line (path, command)
- Output truncated with "… (23 more lines)" — expandable

### 2.3 Code Block Rendering
```
  ┌─ rust ──────────────────────┐
  │ fn main() {                 │
  │     println!("hello");      │
  │ }                           │
  └─────────────────────────────┘
```
- Language tag in header
- Subtle background (slightly lighter than chat bg)
- Line numbers optional (toggle with key)
- Syntax highlighting via simple keyword matching (no external dep needed)

### 2.4 Diff Display
```
  ┌─ diff: src/main.rs ─────────┐
  │ - old line                   │  (red)
  │ + new line                   │  (green)
  │   context line               │  (muted)
  └─────────────────────────────┘
```
- Parse unified diff format
- Color +/- lines
- Show file header

### 2.5 Thinking Animation
- Replace dots with a spinning line character: `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏`
- Or use a pulsing `◉` with color shift
- Show elapsed time: `thinking... 2.3s`

## Phase 3: Input Box Redesign (chat.rs + handler.rs)

### 3.1 Multi-line Input
```
┌──────────────────────────────────────────────────┐
│ > Implement a REST API with authentication       │
│   and rate limiting                               │
│                                                   │
└──────────────────────────────────────────────────┘
```
- Auto-grow up to 5 lines
- Placeholder: `Ask anything...` (muted, italic)
- Cursor: block cursor with primary color
- Character count in bottom-right corner

### 3.2 Command Autocomplete
```
┌──────────────────────────────────────────────────┐
│ > /mo                                             │
├──────────────────────────────────────────────────┤
│  /models     Manage model pool                    │
│  /pool       Manage experience pool               │
└──────────────────────────────────────────────────┘
```
- Popup below input, not above
- Highlighted selection with background
- Preview of command description

### New Design
```
  gpt-4 · ↑2.1k ↓3.4k · pool: 128 · budget: 72%     Ctrl+A providers · / cmds

- Left: model name, token stats (compact), pool count, budget
- Right: context-aware keyboard hints
- No meaningless stats ($0.000, CH0.0%)
- Colors: model=primary, tokens=muted, budget=color-coded by %, hints=muted
